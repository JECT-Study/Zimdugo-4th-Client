# 설정 공지사항 및 정책 문서 설계

## 목표

설정 페이지의 공지사항 메뉴에서 기존 약관 페이지와 동일한 문서 UI를 사용해
서비스 초기 운영 안내를 제공한다. 이용약관과 개인정보처리방침의 임시 검토 안내는
제거하고, 실제 서비스 운영과 제보 기능에 필요한 내용을 보강한다.

## 설계

- `LegalDocumentPage`를 공지사항, 이용약관, 개인정보처리방침의 공통 화면으로 사용한다.
- 문서 모델에서 빨간 임시 안내를 표현하던 `notice` 속성을 제거한다.
- 기존 `/settings/notices` 라우트의 미구현 화면을 공지 문서 화면으로 교체한다.
- 공지사항에는 초기 운영 상태, 보관함 제보 요청, 정보 확인 유의사항, 문의 채널을 담는다.
- 이용약관에는 사용자 제보의 이용 범위, 게시 정보의 변동 가능성, 공지 및 문의 조항을 보강한다.
- 개인정보처리방침에는 제보 사진에 포함된 EXIF 메타데이터의 항목, 처리 목적,
  동의 방식, 보유 및 파기 원칙을 명시한다.

## 검증

- 문서 데이터 단위 테스트로 필수 문구와 임시 안내 제거를 검증한다.
- Web 앱 TypeScript 검사와 Biome 검사를 실행한다.
- 설정 페이지에서 공지사항, 이용약관, 개인정보처리방침을 직접 열어 렌더링과 뒤로가기를 확인한다.

## 범위 제외

- 정책 문서의 다국어 번역
- 공지사항 목록, 서버 연동, 게시일 관리
- 사업자 또는 수탁자 정보 확정
